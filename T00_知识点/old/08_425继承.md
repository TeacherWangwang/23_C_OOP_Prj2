6. 封装(encapsulation):
   1. 把属性与方法(行为)"封装"到一个类中
   2. 隐藏属性与行为(访问修饰符)
       1. public 公有:完全公开
       2. protected 受保护:自己与子类公开
       3. private 私有:只能自己能访问
   3. 隐藏方法实现细节:不提供源代码
7. 实体类(Entity Class)
   1. 属性私有private 数据类型 属性名;
   2. set方法
   void set首字母大写的属性名(数据类型 参数名) {
      类名::属性名 = 参数名;//参数名一般与属性名相同
      //this -> 属性名 = 参数名;
   }
   3. get方法
   数据类型 get首字母大写的属性名() {
      return 属性名;
   }
   4. set有参数没返回值,get有返回值没参数
   5. set与get又称为"访问器"
### 新知识   
8. 继承
   1. 在原有类的基础上定义一个新类叫继承(派生)
   2. "子类是父类"来确定父子关系
   3. 父子类的关系是相对的 
   4. C++是多继承(类的体系结构变的混乱,菱形继承), Java,C#单继承
   5. 语法 子类 : 父类 (java 子类 extends 父类, C#)
   6. 特点
      1. 特点一:子类拥有父类的属性与方法
         1. 反推出什么情况下使用继承,继承使用的场景:
         2. 当两个类中有多个相同的属性或行为时,抽取父类,把相同的属性与行为放到父类中
         3. 减少代码冗余,实现代码复用
      2. 特点二:子类可以拥有自己新的属性与方法
      3. 特点三:子类重写父类的方法(方法重写)
         1. 方法重写(方法覆盖)
         2. 子类方法与父类方法具有相同的方法声明(方法头),不同的方法实现(方法体)
   7. 封装
      1. public 公有:完全公开
      2. protected 受保护:自己与子类可以访问
      3. private 私有:只能自己能访问 
   8. C++继承方式(Java 公有继承)
      1. 公有继承,子类 : public 父类, 属性与方法的访问修饰符不变
      2. 保护继承,子类 : protected 父类, 父类的公有的会变为子类(本类)的保护的
      3. 私有继承,子类 : private 父类, 父类的公有的会变为子类(本类)的私有的
   9. 多继承:菱形继承
      1. 一个类有多个父类,多个父类中有相同的方法
      2. 通过子类调用方法,不知道该调用那个父类的
      3. 解决: 对象名.类名::方法名()

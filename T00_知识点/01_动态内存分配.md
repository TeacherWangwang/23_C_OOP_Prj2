1. 指针
   1. 指针变量：存储另外一块内存的内存地址
   2. 直接写变量名：“访问”它具体内存中存储的值,普通变量存储业务数据，指针变量存储另外一块内存地址
   3. &变量名：“得到”它的内存首地址
   4. *指针变量名： 访问的它指向的内存中存储的数值
2. 栈
   1. 缘起:数据结构,先进后出,后进先出(手枪弹夹子弹,非左轮)
   2. 特点一:系统自动分配内存,系统自动回收内存;什么时候回收:超出变量的作用域;什么是作用域:变量所在的大括号
   3. 特点二:内存可以有变量名(别名)
   4. 访问变量方法:变量名或者指向它的指针
3. 堆
   1. 特点一:程序员自己分配内存,程序员自己回收内存
      1. 如何分配: c语言 malloc(sizeof(数据类型))函数分配,返回分配的内存首地址  c++:new
      2. 如何回收: c语言 free(堆内存地址,栈指针变量中存储)              c++:delete
   2. 特点二:内存没有名字(变量名),只能通过指向它的指针访问
   3. 如果不释放,造成"内存泄漏"    内存溢出=内存不够用
4. 最重要:C,C++ 变量(对象),既可以在栈中,又可以在堆中,Java只能在堆中
5. 栈与堆
   1. C++内存会分为多个不同的逻辑区域:栈,堆等
   2. 栈:系统自动分配,自动回收,不会内存泄漏
      1. 栈中的内存都有逻辑名(变量名)
      2. int a = 10; //普通变量
      3. Student zhangsan; //对象
      4. int *p; Student *s;
      5. 第一种访问方式:变量名.属性名 变量名.方法名()
      6. 第二种访问方式: 声明一个指针变量,指向栈中内存首地址
      7. 类名* 指针变量名; 指针变量名 = &对象名;
      8. 访问:(*指针变量名).属性名  或者  指针变量名->属性名
   3. 堆,程序员手工申请,手工回收,如果没有回收,会造成内存泄漏
      1. 堆中内存不能起别名(变量名),只能通过内存首地址访问它
      2. 如何申请:
         1. c标准: 数据类型* 指针变量 = (数据类型*)malloc(sizeof(数据类型));
         2. C++:数据类型* 指针变量 = new 数据类型();
      3. 如何释放:
         1. c标准: free(指针变量);
         2. C++: delete 指针变量;
      4. 存储堆内存首地址,需要在栈中声明指针变量
      5. 只能通过指针变量访问堆内存
   4. 考试题总结
      1. 类的属性声明为指针类型: 类的属性值在堆中
      2. 构造方法内申请堆内存
      3. 析构方法内释放堆内存
